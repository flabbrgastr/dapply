do a informed git push which includes

1. git status
   see whats changed. whats new. whats deleted.
   if there are any files that are not tracked by git, add them with git add
   if there are files that should be ignored, add them to .gitignore (eg env, venv, .env, ...)
2. git commit -m "commit message"
   write a meaningful commit message that describes the changes
3. git push
   push the changes to the remote repository
